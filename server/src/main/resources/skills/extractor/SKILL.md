# 角色、场景与道具提取指南

## 角色提取规范

**重要：只提取有对话内容的主角和配角，龙套角色不提取！**

**必须满足的提取条件**：
- 角色在剧本中有对话内容
- 角色定位为主角或配角（龙套不提取）
- 每个提取的角色必须生成完整的prompt

提取的角色信息包含：
- **姓名**：角色全名
- **角色定位**：主角/配角（不提取龙套）
- **外貌描写**：性别、年龄、体型、面部特征、发型、着装（300-500字）
- **性格特点**：核心性格标签
- **角色描述**：背景故事和关系
- **prompt**：必须为每个角色生成完整的图片生成提示词

## 场景提取规范

提取的场景/背景信息包含：
- **地点**：具体场所名称
- **时间**：时间段和光线条件
- **氛围**：环境氛围描述
- **提示词**：用于AI图片生成的英文提示词（纯背景，不含人物）

## 道具提取规范

**重要：只提取剧情关键道具，不要提取常见背景物品！**

**必须提取**的关键道具类型：
- 角色标志性随身物品（武器、饰品、工具）
- 剧情推进的核心物件（信件、钥匙、宝物、重要文件）
- 场景中反复出现且有特殊意义的物品
- 具有象征意义或特殊功能的道具

**不要提取**的常见物品：
- 日常家具（桌子、椅子、床、柜子）
- 普通餐具（碗、筷子、杯子、盘子）
- 常见文具（笔、纸、本子）
- 一般装饰品（花瓶、画框、窗帘）
- 普通交通工具（普通汽车、自行车）

提取的道具信息包含：
- **名称**：道具名
- **类型**：武器/饰品/文件/宝物/工具/其他
- **描述**：外观和用途（100-200字）
- **图片提示词**：用于AI图片生成的英文提示词（单独物品，白底或简洁背景）

## 提示词生成要求

### 角色提示词生成（两步）

**步骤 1：生成 appearance（详细外貌）**

基于角色描述，生成详细的外貌特征描述（中文），包含：
- 年龄与体型：年龄段、身高体型、体态特征
- 发型：长度、颜色、造型、质感
- 面部特征：脸型、眉形、眼型、鼻型、唇形
- 肤色与妆容：肤色、妆容风格、重点妆容

要求：具体、视觉化、适合图片生成

**步骤 2：生成 prompt（完整图片提示词）**

基于 description + appearance，生成完整的图片生成提示词（中文），包含：
1. 外貌特征（基于 appearance）
2. 服装细节（面料、颜色、款式、剪裁）
3. 配饰细节
4. 姿态与气质
5. 技术要求：非对称水平排版。左侧：一张占据全图约40%宽度的超大头肩肖像。右侧：三个并排的全身视图（正面、右侧面、背面）。放松A字姿势，完美身份一致性，全身图头部高度统一。纯白背景，主体隔离。均匀柔和的影棚平光，自然阴影无强烈对比。焦点锐利，可打印，8k

输出格式：一段连贯的中文描述，适合直接用于图片生成

### 场景提示词生成

基于场景的 location 和 time，生成完整的环境渲染提示词（中文），包含：
1. 空间布局与构图
2. 光线与氛围（根据时间）
3. 色调与质感
4. 环境细节（家具、装饰、道具）
5. 技术要求：电影级渲染，景深效果，高清细节

输出格式：一段连贯的中文描述

### 道具提示词生成

**重要：只提取对剧情有重要作用的关键道具，剧中多次出现，忽略常见背景物品（如桌椅、杯子、笔等）**

基于道具的 description，生成完整的图片提示词（中文），包含：
1. 道具的材质、颜色、尺寸、造型
2. 细节特征（纹理、装饰、磨损等）
3. 技术要求：白底三视图，包含正面、左侧面、俯视图，物体居中，光线均匀，无阴影

输出格式：一段连贯的中文描述

所有描述使用中文，具体、视觉化、适合图片生成。

## 使用步骤

1. 调用 `read_script_for_extraction` 读取当前集剧本
2. 调用 `read_existing_characters` 查看项目已有角色
3. 调用 `read_existing_scenes` 查看项目已有场景
4. 调用 `read_existing_props` 查看项目已有道具
5. 分析剧本，提取当前集涉及的角色、场景和关键道具
6. 调用 `save_dedup_characters` 保存角色并关联到当前集
7. 调用 `save_dedup_scenes` 保存场景并关联到当前集
8. 调用 `save_dedup_props` 保存道具

**重要：调用工具时JSON格式要求**
- 所有JSON参数必须使用严格的JSON格式
- 字段名必须用双引号包裹，例如 `{"name":"张三"}` 而不是 `{name=张三}`
- 字符串值必须用双引号包裹
- 数组使用方括号 `[]`，对象使用花括号 `{}`

## 当前集规则

- 目标是补齐”当前集”需要的角色、场景和道具，不是重扫整个项目
- 若角色或场景已在项目中存在但当前集未关联，仍应复用并关联到当前集
- 若项目中已有同名角色，优先复用，不要重复创建
- 若场景的location相同，更新time和prompt（因为同一地点在不同时间段需要不同的渲染效果）
- 道具按名称去重，已存在的跳过

## 角色提取质量要求

**每个角色必须包含完整的prompt字段，否则拒绝保存该角色**

调用 `save_dedup_characters` 时，传入的JSON中每个角色对象必须包含：
- name（必填）
- role（必填，且只能是”主角”或”配角”）
- description（必填）
- appearance（必填）
- personality（必填）
- prompt（必填，完整的图片生成提示词）

如果某个角色缺少prompt或prompt为空，不要将该角色加入保存列表。
